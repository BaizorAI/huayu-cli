# PRD：huazhen — 统一打包与一键安装

## Problem Statement

huazhen 是一个 Rust CLI/TUI 编程工作台，将 baizor.com API、Codex 和 Claude Code 统一封装。当前实现已具备登录轮询、隔离配置目录、PTY 子进程执行、slash commands、状态栏、设置弹窗等核心功能，但用户体验存在三个关键缺口：

1. **安装门槛高**：没有一键安装方式；Codex 和 Claude 的工具安装依赖 npm / Node.js，在 Windows 上障碍大，PATH 解析失败率高。
2. **分发不完整**：huazhen 没有发行版打包，无法通过一条命令部署完整工作台（huazhen + codex + claude）。
3. **Windows 终端乱码**：中文 UI 文案、边框符号和状态图标在 Windows 终端输出中出现乱码，影响用户信任和可读性。

## Solution

将 huazhen、codex 和 claude 统一打包进一个发行版，通过一行命令完成 Windows 一键安装：

```powershell
irm https://baizor.com/install/huazhen.ps1 | iex
```

安装完成后，用户直接运行 `huazhen` 即可进入工作台，无需 Node.js、npm 或任何额外依赖。

**核心决策：**

- huazhen GitHub Release 每个版本随附 `huazhen-x86_64-pc-windows-msvc.zip`，压缩包内含 `huazhen.exe`、`tools/codex.exe`、`tools/claude`（或 claude 安装器）。
- PS1 安装脚本托管于 `https://baizor.com/install/huazhen.ps1`，源文件维护于 `C:\Users\Lucky\baizor\baizor-new-api\install\huazhen.ps1`，通过已有部署流程同步到 baizor.com。
- 修复 Windows 控制台代码页，所有 UTF-8 中文文案、边框符号在 Windows PowerShell 和 Windows Terminal 中正确显示。
- 保持现有 TUI/CLI 架构不变；测试覆盖配置、事件解析、命令解析、历史导航和 apply_settings。

## 一键安装体验

### 安装脚本行为

`huazhen.ps1` 按以下步骤执行：

1. 检测 CPU 架构（当前仅支持 x64）。
2. 通过 GitHub API 获取最新 Release 版本号。
3. 从 `github.com/BaizorAI/huazhen/releases` 下载 `huazhen-x86_64-pc-windows-msvc.zip`。
4. 解压到 `%USERPROFILE%\.huazhen\`：
   - `huazhen.exe` → `%USERPROFILE%\.huazhen\bin\`
   - `codex.exe` → `%USERPROFILE%\.huazhen\tools\`
   - `claude*` → `%USERPROFILE%\.huazhen\tools\`
5. 将 `%USERPROFILE%\.huazhen\bin` 追加到 User PATH（持久化到注册表）。
6. 打印后续操作提示。

脚本对错误有完整处理：架构不支持、网络失败、解压失败时给出明确提示后退出；部分工具缺失时打印警告但不中断安装。

### Release 压缩包内容（Windows x64）

```
huazhen-x86_64-pc-windows-msvc.zip
├── huazhen.exe          # huazhen 本体
└── tools/
    ├── codex.exe        # 锁定版本的 codex（Rust native binary）
    ├── codex.version    # 版本号文件，供 is_current_version() 读取
    ├── claude           # claude binary / 安装器
    └── claude.version
```

### 目录结构（安装后）

```
%USERPROFILE%\.huazhen\
├── bin\
│   └── huazhen.exe
├── tools\
│   ├── codex.exe
│   ├── codex.version
│   ├── claude
│   └── claude.version
├── codex\               # codex 隔离配置（CODEX_HOME）
│   ├── config.toml
│   └── auth.json
├── claude\              # claude 隔离配置（CLAUDE_CONFIG_DIR）
│   └── settings.json
└── config.json          # huazhen 主配置
```

## User Stories

### 安装与分发

1. As a Windows user, I want to run one command (`irm https://baizor.com/install/huazhen.ps1 | iex`) and have huazhen, codex, and claude fully installed, so that I can start coding without installing Node.js, npm, or any prerequisite.
2. As a Windows user, I want the installer to add huazhen to my PATH automatically, so that I can run `huazhen` from any terminal immediately after install.
3. As a user, I want the installer to show clear progress and confirm what was installed, so that I know the install succeeded.
4. As a user, I want the installer to warn but not fail when a bundled tool is missing from the archive, so that a partial install does not block the rest.
5. As a developer, I want the PS1 script source to live at `C:\Users\Lucky\baizor\baizor-new-api\install\huazhen.ps1`, so that it is version-controlled alongside the other repo assets and deployed to `https://baizor.com/install/`.

### Windows 编码修复

6. As a Windows user, I want all Chinese characters, box-drawing symbols, and status icons to display correctly in PowerShell and Windows Terminal, so that the TUI feels trustworthy.

### 新布局与键盘交互

7. As a user, I want the left panel to show all execution output (logs, AI replies, events, errors) in a single scrollable view, so that I do not have to look in two places at once.
8. As a user, I want the right panel to show a static help reference (shortcuts, current tool/model/working directory, recent commands), so that I can check any shortcut without interrupting the task in progress.
9. As a user, I want pressing Up/Down to navigate input history, so that I can re-send or edit a previous prompt.
10. As a user, I want Alt+Q to quit, so that `q` alone never exits accidentally.
11. As a user, I want Enter to send input or confirm an overlay, and Esc to cancel/close.
12. As a user, I want Page Up/Down and mouse wheel to scroll the main panel.
13. As a user, I want Tab to cycle tools (codex ↔ claude) from the status bar.

### 登录与首次配置

14. As a new user, I want the app to open a login overlay when no API Key exists.
15. As a new user, I want the login URL to be readable and copyable from the overlay.
16. As a returning user, I want saved config to load automatically on start.

### 状态与配置

17. As a user, I want `huazhen status` to show API Key (masked), base URL, model, active tool, and installed tool versions.
18. As a user, I want `/model <name>` to update and persist the default model.
19. As a user, I want a settings overlay (`s` key) for model and base URL.

### 捆绑工具与更新

20. As a user, I want `/update` or `huazhen update` to download the latest compatible tool versions.
21. As a user, I want download progress streamed into the main output panel during `/update`.
22. As a power user, I want huazhen to fall back to PATH-installed tools when bundled tools are absent.
23. As a user, I want a startup warning when bundled tools are outdated.

### 任务执行

24. As a user, I want pressing Enter on plain text to spawn the active tool.
25. As a user, I want Esc to cancel the running task.
26. As a user, I want task completion to show elapsed time.
27. As a user, I want authentication errors to update status and suggest login.
28. As a user, I want missing tool errors to suggest `/update`.

### 开发者

29. As a developer, I want the PS1 script to be self-contained and deployable via `irm | iex` with no external dependencies.
30. As a developer, I want tests around config isolation, event parsing, command parsing, binary resolution, and app state.
31. As a developer, I want CI-friendly tests that redirect config paths to temporary directories.

## Implementation Decisions

### PS1 安装脚本（`C:\Users\Lucky\baizor\baizor-new-api\install\huazhen.ps1`）

- **托管路径**：源文件 `C:\Users\Lucky\baizor\baizor-new-api\install\huazhen.ps1`，部署后通过 `https://baizor.com/install/huazhen.ps1` 访问。
- **使用方式**：`irm https://baizor.com/install/huazhen.ps1 | iex`（PowerShell 5.1+）。
- **平台**：仅支持 Windows x64（`PROCESSOR_ARCHITECTURE == AMD64`）；其他架构直接报错退出。
- **版本获取**：GET `https://baizor.com/install/version.txt`，纯文本版本号（如 `v0.1.5`），不依赖 GitHub API。
- **下载目标**：`https://baizor.com/install/huazhen-x86_64-pc-windows-msvc.zip`（baizor.com 直接提供，不走 GitHub）。
- **安装目录**：
  - `%USERPROFILE%\.huazhen\bin\huazhen.exe`（本体）
  - `%USERPROFILE%\.huazhen\tools\codex.exe`（捆绑 codex）
  - `%USERPROFILE%\.huazhen\tools\claude*`（捆绑 claude）
- **PATH 修改**：写入 User 级注册表（`[Environment]::SetEnvironmentVariable(..., "User")`），同时更新当前会话 `$env:PATH` 使脚本内 `huazhen` 命令立即可用。
- **错误处理**：网络失败、解压失败报错退出；工具文件缺失打警告后继续；最终打印 `huazhen update` 提示。
- **不需要管理员权限**：安装到 User Profile 目录，PATH 写 User 级，无需 elevation。

### Release 打包约定

每次 huazhen 发布时，CI 构建产物上传到 `https://baizor.com/install/`：

| 文件 | 说明 |
|------|------|
| `version.txt` | 最新版本号（纯文本，如 `v0.1.5`） |
| `huazhen-x86_64-pc-windows-msvc.zip` | Windows x64 完整包 |
| `huazhen-x86_64-apple-darwin.tar.gz` | macOS Intel 完整包 |
| `huazhen-aarch64-apple-darwin.tar.gz` | macOS Apple Silicon 完整包 |
| `huazhen-x86_64-unknown-linux-musl.tar.gz` | Linux x64 完整包 |

每个压缩包内部结构统一：根目录 `huazhen`（或 `huazhen.exe`）+ `tools/` 子目录含锁定版本工具。

### 捆绑工具策略

- codex：从 `https://baizor.com/install/` 下载预置的 native binary，当前锁定 `0.142.5`（CI 构建时从 openai/codex GitHub Release 拉取后再转存到 baizor.com）。
- claude：当 Anthropic 发布 native binary 时捆绑；在此之前，Release zip 包含 `tools/claude-npm-setup.cmd`，首次启动时自动执行 npm 安装（仍需 Node.js），或 user 手动 `/update claude`。
- 工具解析优先级：`~/.huazhen/tools/` 捆绑版 > 系统 PATH（与现有 `local_binary` + `which::which` 逻辑一致）。
- `~/.huazhen/tools/<name>.version` 文件记录已安装版本，供 `is_current_version()` 对比。

### Windows 编码修复

- `main()` 最开头加 `#[cfg(windows)]` 块，通过 `extern "system"` 调用 `SetConsoleOutputCP(65001)` 和 `SetConsoleCP(65001)`，切换控制台代码页到 UTF-8。
- 不添加新依赖（kernel32 原生调用）。
- 不影响 macOS/Linux（整个块被编译器剔除）。

### 界面布局（保持现有实现）

- 左主工作区（~70%）：所有执行输出合并在单一滚动视图。
- 右静态帮助面板（~30%）：快捷键速查 + 当前状态 + 最近 slash 命令。
- 固定 70/30 分割，不可调。

### 键盘映射

| 按键 | 行为 |
|------|------|
| `Enter` | 发送输入 / 弹窗确认 |
| `Esc` | 取消任务 / 关闭弹窗 |
| `↑/↓` | 输入历史导航（语义单一） |
| `Page Up/Down` | 主面板翻屏（↑暂停自动滚动，↓到底恢复） |
| 滚轮 | 上下滚动 |
| `Alt+Q` | 退出（不使用单键 `q`） |
| `Tab` | 切换工具（codex ↔ claude） |
| `Shift+Enter` | 输入框换行 |
| `s`（输入为空） | 打开设置弹窗 |

### 其余约定

- Rust CLI/TUI 技术栈：clap、ratatui/crossterm、tokio/reqwest、portable-pty。
- 单一 huazhen 配置根目录（`~/.huazhen/`），可通过 `HUAZHEN_CONFIG_DIR` 环境变量覆盖（测试隔离 seam）。
- Codex 配置通过 `CODEX_HOME` 注入，Claude 配置通过 `CLAUDE_CONFIG_DIR` 注入。
- 安装/更新通过 channel 异步向主工作区流式输出进度，以 `__DONE__` 哨兵结束。

## Testing Decisions

好的测试断言外部可观察行为和稳定契约，不测 ratatui 像素或私有实现细节。

**已覆盖（56 个测试，`cargo test` 全部通过）：**

- Config defaults、save/load round-trip、codex/claude 配置文件内容、文件隔离。
- API key masking。
- 全部 slash 命令解析（login/switch/model/update/install/status/help/?/clear/quit/exit/q/unknown），非命令返回 None，UpdateTarget::tool_names。
- parse_event：auth 错误（401/invalid key/auth failed）、network 错误、file written、test pass/fail、normal line 直通。
- local_binary：bundled 优先于 PATH、tools_dir 为空/不存在时返回 None、version 匹配/过期/缺失、Windows `.exe` 后缀。
- App 输入历史 navigation（up from empty、cycle、down past end restores draft）。
- App submit 行为（empty、not logged in、/help、/clear）。
- apply_settings 更新内存并关闭弹窗。

**待补充测试：**

- PS1 安装脚本：由 PowerShell Pester 框架测试（独立于 Rust 测试，CI 单独步骤）。
- login polling：mocked HTTP success/pending/network failure/timeout。
- model fetch：mocked `/v1/models` success 和 parse failure。
- 工具下载进度 channel：`__DONE__` 哨兵、HTTP 错误的进度行内容。

**测试 seam：**

- `HUAZHEN_CONFIG_DIR` 环境变量（共享 `CONFIG_LOCK` + `TempConfigGuard`，避免并行测试竞争）。
- Command parser seam for slash commands。
- Tool event parser seam for progress classification。
- App state methods for TUI behavior without rendering a terminal。
- Tool downloader progress channel seam。

## Out of Scope

- macOS / Linux 一键安装脚本（当前只做 Windows PS1；其他平台手动下载 tar.gz）。
- Building a web UI。
- Adding Gemini, OpenCode, Hermes, or other AI coding tools。
- Persistent chat history across app restarts。
- Multi-account profile management。
- Automatic self-update of the huazhen binary itself（用户重新运行 PS1 更新 huazhen）。
- Full terminal snapshot testing of ratatui rendering。
- Managing the user's system Node.js or npm installation。
- Changing baizor.com server-side API behavior。
- Mouse drag split-pane resizing。
- Supporting npm-installed codex/claude as a first-class path（PATH fallback handles this passively）。

## Further Notes

- **工具版本锁定**：`codex` 锁定 `0.142.5`（Responses API + `--dangerously-bypass-approvals-and-sandbox`）；`claude` 锁定最小支持版本（`--print` + `--dangerously-skip-permissions` 可用）。版本号在 CI 兼容测试验证后才更新，需要 `BAIZOR_TEST_API_KEY`。
- **PS1 脚本不需要管理员权限**：安装到 User Profile，PATH 写 User 级注册表，适合企业环境中没有管理员权限的用户。
- **irm | iex 安全性**：脚本从 baizor.com 域名签发的 HTTPS 端点加载，TLS 1.2+，不执行任何系统级操作（不写注册表 HKLM，不修改系统 PATH，不安装服务）。
- **最近命令只记录 slash 命令**（不含自由文本 prompt），避免泄露用户业务内容。
- **`cargo test` 当前状态**：56 个测试全部通过，覆盖配置、命令解析、事件解析、二进制解析和 App 状态机。
