# PRD：huazhen 2026-07-05 稳定化与补齐

## Problem Statement

huazhen 已经从早期脚手架推进到可运行的 Rust CLI/TUI 工作台：用户可以通过 baizor.com 登录获取 API Key，在全屏 TUI 中输入需求，并由 huazhen 调起 Codex 或 Claude Code 子进程执行任务。当前代码已经具备登录轮询、隔离配置目录、本地 npm 工具安装、状态栏、设置弹窗、slash commands、PTY 子进程执行和基础进度流展示。

但当前实现仍处在"能跑起来"的阶段，距离稳定可交付还有明显缺口：

- 中文 UI 文案、边框符号和状态图标在 Windows 终端输出中出现乱码，影响用户信任和可读性。
- 既有计划中的 models/install 等命令没有完全暴露为 CLI 子命令，部分服务代码已经存在但没有接入主流程。
- Codex 和 Claude 的配置写入策略不完全一致，Claude 隔离配置写入函数存在但尚未被实际调用。
- TUI 与 CLI 的功能边界不够一致，同一个能力在不同入口下的行为、提示和错误处理存在差异。
- 工具子进程执行、事件解析、安装流程、配置隔离和 TUI 状态迁移缺少足够测试，回归风险偏高。
- 当前只有登录 token/URL 相关单元测试，无法保护主要用户路径。
- **界面布局存在干扰问题**：当前左右两个面板均为流式输出，用户在阅读执行日志时无法快速获取操作参考；上下键同时控制滚动和输入，操作与输入互相干扰；`q` 绑定退出过于靠近常用输入，容易误触。
- **工具依赖链过长**：当前 `/install` 方案通过 npm 安装 codex/claude，要求用户本机已安装 Node.js，在 Windows 上配置 Node.js 环境障碍较高，且 npm 全局安装路径在不同系统上不一致，导致 PATH 解析失败率较高。

## Solution

本 PRD 要求把 huazhen 2026-07-05 当前实现收敛成一个稳定的本地 AI 编程工作台：

- **重新定义界面布局**：左主工作区承载所有执行输出（进度日志+结构化结果合并为单一主视图），右侧改为静态帮助参考面板（快捷键速查、当前工具/模型状态、最近命令记录），右面板不参与执行输出，消除两列竞争注意力的问题。
- **统一键盘映射，消除操作与输入的干扰**：输入框获焦时 ↑/↓ 导航历史输入；输入框为空时 ↑/↓ 滚动左主工作区；退出改为 `Alt+Q`，避免 `q` 误触；`Enter` 用于发送或弹窗确认；`Esc` 取消任务或关闭弹窗；`/help` 查看完整命令列表。
- **将 codex 和 claude 以预编译二进制捆绑进 huazhen 发行版**：huazhen 下载安装后即附带固定版本的工具，无需 Node.js 或 npm，解压即可运行；工具文件提取到 `~/.huazhen/tools/`；power user 如有自己的 PATH 安装版本，优先使用捆绑版，PATH 作为降级回退。
- 修复所有用户可见中文文案、图标和边框符号的编码显示问题，确保 Windows PowerShell、Windows Terminal、macOS/Linux 终端中都能读。
- 补齐 CLI 与 TUI 的功能一致性：登录、状态、模型查看/选择、工具更新、工具切换、设置保存都应有清晰入口。
- 统一配置隔离策略：huazhen 只写入自己的配置根目录，Codex 和 Claude 均通过隔离目录与环境变量读取配置。
- 将已存在但未接入的模型拉取、Claude 配置写入、错误类型和事件分支纳入真实流程，或移除无用死分支。
- 强化工具执行体验：运行中可取消、可继续向 PTY 输入确认内容、可看到任务耗时、认证失败和网络失败有明确提示。
- 建立围绕外部可观察行为的测试网，重点覆盖配置文件结果、命令输出、状态迁移、工具事件解析和安装进度事件。

## 界面布局

```
+-- huazhen | codex [v] | huazhen-fable-5 | [connected] -------------------+
|                                                                            |
+----------------------------------------+-----------------------------------+
|  主工作区 (左, ~70%)                    |  帮助与参考 (右, ~30%, 静态)          |
|                                        |                                   |
|  执行日志、AI 输出、文件事件、            |  快捷键速查：                        |
|  错误提示、任务完成耗时                  |    Enter   发送 / 弹窗确认           |
|  合并展示在同一滚动视图中                 |    Esc     取消任务 / 关闭弹窗       |
|                                        |    Up/Down  输入历史导航            |
|  内容从顶部开始填充；                    |    PgUp    向上翻屏（暂停自动滚动）   |
|  满屏后新内容追加到底部，                 |    PgDn    向下翻屏 / 回到底部       |
|  视图自动钉在最新内容；                   |    滚轮    上下滚动                  |
|  向上翻页后自动滚动暂停，                 |    Alt+Q   退出                     |
|  PgDn 回底部后自动滚动恢复               |    /help   完整命令列表              |
|                                        |                                   |
|  [任务运行时边框高亮]                    |  当前状态：                         |
|                                        |    工具    codex                  |
|                                        |    模型    huazhen-fable-5        |
|                                        |    目录    /path/to/cwd           |
|                                        |                                   |
|                                        |  最近命令：                         |
|                                        |    /model gpt-5.5                 |
|                                        |    /install codex                 |
+----------------------------------------+-----------------------------------+
|  > 输入需求或 /command_                                                      |
+----------------------------------------------------------------------------+
```

**布局设计原则：**

- 左主工作区：唯一的动态输出区域，所有运行日志、AI 回复、错误信息统一在此显示；不再拆分"输出"和"进度"两列。内容从顶部开始填充，满屏后新内容追加到底部，视图自动钉在最新内容（等同于 `tail -f`）。
- 右帮助区：完全静态，不随任务执行更新（除工具/模型/目录状态字段外）；用户随时可以看到快捷键而不需要离开主界面。
- 输入框：单行显示当前输入，`Shift+Enter` 换行支持多行 prompt；`↑/↓` 始终只做输入历史导航（语义单一，不兼做滚动）。
- 滚动由 Page Up/Down 和鼠标滚轮独占：向上翻页时自动滚动暂停，Page Down 回到底部后自动滚动恢复。
- 状态栏：顶部单行，显示工具选择器（可按 `Tab` 切换）、当前模型、连接状态，不占用主工作区空间。

## User Stories

### 新布局与键盘交互

1. As a user, I want the left panel to show all execution output (logs, AI replies, events, errors) in a single scrollable view, so that I do not have to look in two places at once.
2. As a user, I want the right panel to show a static help reference (shortcuts, current tool/model/working directory, recent commands), so that I can check any shortcut without interrupting the task in progress.
3. As a user, I want pressing Up/Down in the input box when it has text to navigate input history, so that I can re-send or edit a previous prompt without retyping.
4. As a user, I want pressing Up/Down in the input box when it is empty to scroll the main output panel, so that scrolling and history navigation never interfere with each other.
5. As a user, I want Alt+Q to quit, so that the quit action requires a deliberate chord and cannot be triggered by accidentally pressing `q`.
6. As a user, I want Enter to send the input or confirm the active overlay prompt, so that a single key covers both send and confirm without ambiguity.
7. As a user, I want Esc to cancel the running task or close the active overlay, so that one key covers all "back out" actions.
8. As a user, I want `/help` to show the full command list in the main output panel, so that help is always reachable from the input box.
9. As a user, I want the right panel to show the current working directory, so that I can confirm codex is operating in the right project.
10. As a user, I want Tab in the status bar to cycle through available tools (codex / claude), so that switching tools does not require a slash command.
11. As a user, I want the main output panel border to highlight when a task is running, so that I have a passive signal about execution state.
12. As a user, I want Shift+Enter in the input box to insert a newline, so that I can write multi-line prompts without accidentally sending.

### 登录与首次配置

13. As a new huazhen user, I want the app to open a login overlay when no API Key exists, so that I can start without reading setup docs.
14. As a new huazhen user, I want the login URL to be readable and copyable from the overlay, so that I can finish browser login quickly.
15. As a new huazhen user, I want login timeout and network failures to show clear messages, so that I know whether to retry or check connectivity.
16. As a returning huazhen user, I want saved config to load automatically, so that I can start working immediately.

17. As a user, I want the status bar to show active tool, model, and connection state, so that I understand the current execution context.
18. As a user, I want all UI text to render correctly on Windows and macOS/Linux terminals, so that the TUI feels trustworthy and usable.
19. As a user, I want to run `huazhen status`, so that I can inspect API Key presence, base URL, default model, active tool, and installed tool availability.
20. As a user, I want masked API Key display in status, so that output is useful without leaking secrets.
21. As a user, I want `huazhen login` to save the same config used by the TUI.
22. As a user, I want model list fetching to be available from a slash command.
23. As a user, I want `/model <name>` to update and persist the default model.
24. As a user, I want a settings overlay (`s` key) for model and base URL.

### 捆绑工具与更新

25. As a user, I want huazhen to ship with bundled codex and claude binaries at pinned compatible versions, so that I can run huazhen without installing Node.js, npm, or any other prerequisite.
26. As a user, I want bundled tool binaries to be extracted to `~/.huazhen/tools/` on first launch, so that they are ready to use without any manual install step.

30. As a user, I want a startup warning when bundled tools are outdated, so that I know when to run `/update`.

### 工具切换

31. As a user, I want `/switch codex` and `/switch claude` to change the active tool.
32. As a user, I want switching tools to cancel an active task cleanly.
33. As a user, I want my input draft to survive tool switching.

## Implementation Decisions

### 捆绑工具分发

- huazhen GitHub Release 的每个版本包含平台对应的工具压缩包（如 `codex-0.142.5-x86_64-pc-windows-msvc.zip`、`claude-1.x.x-x86_64-apple-darwin.tar.gz`）。
- huazhen 二进制内嵌一份**版本清单**（`tools_manifest.json`），记录当前发行版捆绑的 codex/claude 版本号及各平台下载 URL。
- **首次启动**：若 `~/.huazhen/tools/` 下未检测到对应工具，自动从清单 URL 下载并解压，进度流式输出到主工作区。
- **工具解析优先级**：捆绑目录 `~/.huazhen/tools/` > 系统 PATH。先检查捆绑目录，找到则使用；找不到时才回退到 PATH，允许 power user 使用自己安装的版本。
- **`/update` 命令**（或 `huazhen update` CLI 子命令）：从 GitHub Release API 查询最新清单，与本地已安装版本对比，下载有差异的工具包并替换。
- **启动版本检查**：与清单中的版本对比，若 `~/.huazhen/tools/` 中的工具版本低于清单推荐版本，在主工作区显示一次提示（`[warn] codex 版本 0.x.x 已过期，运行 /update 获取 0.142.5`），不阻塞启动。
- 版本清单随 huazhen 发行版一同发布，锁定已通过 CI 兼容测试的工具版本，保证主版本内工具行为一致；huazhen 升级可带入新清单，允许跨大版本切换工具。
- 移除 `installer.rs` 中基于 npm 的安装逻辑，改为 HTTP 下载 + 解压；进度 channel 接口保持不变（`__DONE__` 哨兵），测试 seam 无须调整。

### 界面布局

- 合并当前的左侧"输出面板"和右侧"进度面板"为单一**主工作区**（左，约占 70% 宽度），所有执行事件、AI 回复、文件写入、错误均追加到同一列表中，保留自动滚动与暂停逻辑。
- 右侧改为**静态帮助参考面板**（约 30% 宽度），内容分三块：快捷键速查（固定文案）、当前状态（工具名、模型名、当前工作目录，随配置变化更新）、最近命令历史（最多显示最近 5 条 slash 命令）。右面板不参与执行输出，不自动滚动。
- 移除 `split_ratio` 可调逻辑，简化为固定 70/30 分割，减少 ↑/↓ 键的多重含义。

### 键盘映射

| 按键 | 行为 | 条件 |
|------|------|------|
| `Enter` | 发送输入 / 弹窗确认 | 主界面 / 覆盖层 |
| `Esc` | 取消任务 / 关闭弹窗 | 任务运行中 / 覆盖层打开时 |
| `↑` | 输入历史上一条 | 始终（语义单一） |
| `↓` | 输入历史下一条 | 始终（语义单一） |
| `Page Up` | 向上翻屏，暂停自动滚动 | 任意 |
| `Page Down` | 向下翻屏；到底时恢复自动滚动 | 任意 |
| 鼠标滚轮 | 上下滚动，向上时暂停自动滚动 | 任意 |
| `Alt+Q` | 退出程序 | 任意状态 |
| `Shift+Enter` | 输入框换行 | 主界面 |
| `Tab` | 状态栏切换工具 (codex ↔ claude) | 无弹窗时 |
| `s` | 打开设置弹窗 | 输入框为空 |
| `Space` | 切换自动滚动开/关 | 输入框为空 |

- 移除单键 `q` 退出，改为 `Alt+Q`，避免输入字母 `q` 时意外退出。
- `↑/↓` 语义变为单一的输入历史导航，不再兼做滚动控制，彻底消除双语义歧义。
- 滚动由 `Page Up/Down` 和鼠标滚轮独占，启用 crossterm mouse capture 支持滚轮事件。

### 主工作区滚动实现

- 内部维护全量行缓冲 `Vec<String>`（上限可配置，如 10000 行，超出时丢弃最旧行）。
- 渲染时计算可见窗口：`visible_start = max(0, total_lines - viewport_height - scroll_offset)`。
- `scroll_offset = 0`：视图自动钉在最新内容底部（`tail -f` 行为），新行到达时无需任何操作。
- `Page Up`：`scroll_offset += viewport_height`，`auto_scroll = false`。
- `Page Down`：`scroll_offset = max(0, scroll_offset - viewport_height)`；若归零则 `auto_scroll = true`。
- 鼠标滚轮：每格 `scroll_offset ±= 3`，向上时 `auto_scroll = false`，到底时 `auto_scroll = true`。
- 新内容到达时：若 `auto_scroll` 为 true，`scroll_offset` 强制保持 0；否则不修改（用户正在阅读历史，不打断）。

### 输入历史

- 维护一个 `Vec<String>` 作为输入历史，最多保留 50 条，运行中的会话内有效（不持久化到磁盘）。
- 每次 `Enter` 发送后将非空输入追加到历史末尾。
- `↑` 从最新向最旧遍历；`↓` 向最新方向遍历；到达边界后停止（不循环）。
- 进入历史导航时将当前输入暂存，按 `↓` 到达末尾后恢复暂存内容。

### 其余约定（保持不变）

- 保持 Rust CLI/TUI 技术栈：clap、ratatui/crossterm、tokio/reqwest、portable-pty。
- TUI 为主要用户体验，CLI 子命令共享相同配置和服务逻辑。
- 所有用户可见字符串使用有效 UTF-8 源码直接写入，避免依赖终端特定渲染；Windows 下可选 ASCII 替代方案。
- 单一 huazhen 配置根目录，可通过环境变量覆盖用于测试。
- Codex 配置仅写入 huazhen 隔离目录，通过 `CODEX_HOME` 注入；Claude 配置仅写入 huazhen 隔离目录，通过 `CLAUDE_CONFIG_DIR` 注入。
- 安装/更新操作通过 channel 异步向主工作区流式输出进度，以 `__DONE__` 哨兵结束。
- slash 命令解析独立于自由文本 prompt，无终端依赖，便于测试。
- 事件解析保持启发式保守原则：识别认证错误、网络错误、文件写入、测试结果、进程完成，不过度分类普通输出。
- 不在本 PRD 中增加数据库、持久化会话历史、新 AI 工具或 Web 界面。

## Testing Decisions

好的测试应断言外部可观察行为和稳定契约，而非 ratatui 像素输出或私有实现细节。

- Test config defaults: empty config loads base URL, default model, and default tool correctly.
- Test config save/load round trip using an overridden config root.
- Test config isolation: Codex and Claude config writers create files only under the huazhen config root.
- Test Codex config content: selected model and API key are written in the expected managed format.
- Test Claude config content: auth token, base URL with `/v1`, and model are written in the expected managed format.
- Test API Key masking for short keys and normal keys.
- Test login token generation and login URL formatting.
- Test login polling response parsing with mocked success, pending, malformed response, network failure, and timeout paths.
- Test model fetching with mocked `/v1/models` success and parse failure.
- Test slash command parsing for login, switch, model, update, status, help, clear, quit, unknown command, and non-command input.
- Test install target parsing for Codex, Claude, and all.
- Test local binary resolution: bundled path preferred over PATH; PATH fallback when bundled absent; Windows `.exe` suffix handling.
- Test bundled tool extraction: correct target directory, version file written, permission bits set.
- Test startup version check: newer version available triggers warning; same version is silent; missing tools triggers auto-download prompt.
- Test event parsing for auth errors, network errors, file writes, test pass, test failure, and normal log lines.
- Test app submit behavior when not logged in.
- Test app submit behavior for slash commands.
- Test settings application.
- Test install progress drain.
- Test login overlay polling.
- **Test input history navigation**: up from empty adds nothing; up from populated cycles correctly; down past end restores draft.
- **Test keyboard dispatch**: Alt+Q triggers quit; `q` alone does not quit when input box is active.

Existing seams to prefer:
- Command parser seam for slash commands.
- Config load/save and tool-specific config writer seam for file behavior.
- Login/model service seam for HTTP behavior.
- Tool event parser seam for progress classification.
- App state methods for TUI behavior without rendering a terminal.
- Tool downloader progress channel seam for download/update behavior (replaces npm install seam).

## Out of Scope

- Building a web UI.
- Adding Gemini, OpenCode, Hermes, or other AI coding tools.
- Persistent chat history across app restarts.
- Multi-account profile management.
- Cloud sync, WebDAV backup, or config import/export.
- Replacing Codex or Claude with a native LLM client.
- Full terminal snapshot testing of ratatui rendering.
- Automatic self-update of the huazhen binary itself.
- Publishing binaries or packaging installers for huazhen itself.
- Managing the user's system Node.js or npm installation.
- Supporting npm-installed codex/claude as a first-class path (PATH fallback handles this passively).
- Changing baizor.com server-side API behavior.
- Mouse drag split-pane resizing（移除可调分割比例，改为固定 70/30）.

## Further Notes

- 当前 `cargo test` 通过 3 个测试，均围绕登录 token/URL 行为，二进制有多个未使用代码路径的警告。这些警告是本 PRD 的有效信号：要么接入意图功能，要么移除过时分支。
- 最大的产品风险不是 PTY 执行本身，而是文档行为、TUI 可见行为、CLI 命令和隔离配置副作用之间的漂移。
- 键盘映射变化的实现优先级：`Alt+Q` 替换 `q` 退出 → `↑/↓` 双语义（历史/滚动）→ 右面板改为静态帮助 → 输入历史追加逻辑。每一步可独立提交和测试。
- 右侧静态帮助面板的"最近命令"只显示 slash 命令（不包含自由文本 prompt），避免泄露用户的业务 prompt 内容。
- 如果 ratatui 版本升级，优先保持键盘事件处理的正确性，其次才是渲染优化。
- **工具版本锁定策略**：`tools_manifest.json` 中的版本号在 huazhen 发布时人工确认，并通过 CI 集成测试（使用固定版本的工具对 baizor.com 运行 smoke test）验证兼容性后才合入；不自动追踪最新版，避免上游工具 breaking change 导致捆绑版本不可用。codex 当前锁定 0.142.5（Responses API + `--dangerously-bypass-approvals-and-sandbox`），claude 当前锁定需验证的最小支持版本（`--print` + `--dangerously-skip-permissions` 可用）。
- 捆绑工具的 CI 集成测试只在有 `BAIZOR_TEST_API_KEY` 环境变量时运行，本地开发和 PR 检查可跳过，避免消耗 quota。
